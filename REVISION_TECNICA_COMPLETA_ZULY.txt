1. NUCLEO CENTRAL: core/agent.py
═══════════════════════════════════════════════════════════════════════════════

🧠 ¿QUÉ ES?
───────────
Cerebro central de ZULY que orquesta toda la lógica del sistema.
Punto de entrada para todas las peticiones del usuario.

📊 ESTRUCTURA INTERNA:
─────────────────────
class Agent:
  ├─ engine_adapter (EngineAdapter) - Ejecutor de operaciones
  ├─ command_loader (CommandLoader) - Carga comandos disponibles
  ├─ intent_router (IntentRouter) - Enrutador de 48 handlers
  ├─ nlu (NaturalLanguageProcessor) - Detector de intenciones
  ├─ scene_monitor (SceneMonitor) - Monitor de estado Blender
  ├─ pattern_memory (PatternMemory) - Gestor de patrones aprendidos
  ├─ validators (V0, V1, V2, V3) - Capas de validación
  ├─ cognition (CognitionCore) - Sistema de evaluación autónoma
  ├─ blender_observer (BlenderObserver) - Monitor de escena pasivo
  ├─ failsafe_executor (FailsafeExecutor) - Modo seguro
  ├─ action_logger (ActionLogger) - Trazabilidad
  ├─ trace_core (TraceCore) - Auditoría de decisiones
  ├─ context (ExecutionContext) - Estado de sesión
  └─ session_history[] - Histórico de 100+ comandos (rolling window)

🎯 CAPACIDADES PRINCIPALES:
──────────────────────────
✅ process_natural_request(query: str) 
   └─ Entry point principal
   └─ Input: "Crear un cubo dorado"
   └─ Output: Cubo creado en Blender + validación

✅ execute_via_router(command, params)
   └─ Ejecuta handler específico
   └─ Fallback a agente autónomo si falla

✅ get_blender_snapshot()
   └─ Captura estado de Blender
   └─ Rolling window de últimos 5 snapshots

✅ get_context_summary()
   └─ Estado actual de sesión
   └─ Éxitos, fallos, memoria usada

📈 ESTADO ACTUAL:
─────────────────
• 48 handlers Blender registrados ✅
• 4 capas de validación activas ✅
• Pattern memory con 1+ patrones ✅
• Cognition core operativo ✅
• BlenderAdapter activo (si bpy disponible) ✅
• MockAdapter fallback (si no existe bpy) ✅

⚠️ LIMITACIONES:
────────────────
• Solo Blender (no soporta otros engines)
• Max 50 acciones/sesión (límite de complejidad)
• Rolling window: pierde historial antiguo
• No hay persistencia automática de sesión

🔧 ANATOMÍA DE UN HANDLER:
─────────────────────────
def create_cube_handler(parameters: Dict, adapter=None):
    """Ejemplo: Handler para crear cubo"""
    
    # 1. Obtener adapter si no se proporciona
    if adapter is None:
        from core.adapters import get_engine_adapter
        adapter = get_engine_adapter()
    
    # 2. Validar parámetros
    location = parameters.get('location', [0, 0, 0])
    scale = parameters.get('scale', 1.0)
    color = parameters.get('color', [1.0, 1.0, 1.0])
    
    # 3. Ejecutar vía adapter (automáticamente BlenderAdapter o MockAdapter)
    result = adapter.create_primitive('cube', location=location, scale=scale)
    
    # 4. Retornar resultado estructurado
    return {
        'success': True,
        'object': 'Cube.001',
        'message': f'Cubo creado en {location}'
    }

4. SISTEMA DE VALIDACIÓN: core/validation/
═══════════════════════════════════════════════════════════════════════════════

🛡️ ¿QUÉ VALIDA?
──────────────
Capa de verificación en 4 niveles progresivos.

NIVEL V0: Validación de Existencia Física
───────────────────────────────────────────
✅ ¿Existe el objeto en la escena?
✅ ¿Está en las colecciones correctas?
✅ ¿Tiene los parámetros básicos?

Archivo: v0_validator.py
Modo: CRÍTICO (sin pasar V0 no continúa)
Verificación: adapter.get_scene_info()

NIVEL V1: Integridad Estructural
──────────────────────────────────
✅ ¿Tiene la topología correcta?
✅ ¿Los vértices/caras son válidos?
✅ ¿Las normales están correctas?
✅ ¿No hay geometría deformada?

Archivo: v1_validator.py
Modo: VERIFICACIÓN (puede ser pasivo)
Verificación: Análisis profundo de malla

NIVEL V2: Contexto Ambiental
──────────────────────────────
✅ ¿No hay conflictos con otros objetos?
✅ ¿Las propiedades encajan en el contexto?
✅ ¿Hay colisiones no esperadas?
✅ ¿El objeto tiene materiales válidos?

Archivo: v2_validator.py
Modo: GUARDIÁN PRE-EJECUCIÓN
Verificación: Análisis contextual profundo

NIVEL V3: Validación Contextual Avanzada
──────────────────────────────────────────
✅ ¿El usuario realmente quiso esto?
✅ ¿Hay implicaciones ocultas?
✅ ¿Necesita confirmación especial?
✅ ¿Hay dependencias no resueltas?

Archivo: v3_validator.py
Modo: AUDITORÍA (solo logging)
Verificación: Análisis de consecuencias

   ├─ ¿Existe patrón aprendido?
   │   ├─ SÍ → usar_patron
   │   │   └─ Devolver patrón + handler
   │   │
   │   └─ NO → usar_agente
   │       └─ Devolver "usa NLU autónomo"
3. pattern_to_handler_mapper.execute_handler()
   └─ Ejecutar handler con adapter
4. Validación + Aprendizaje

PARÁMETROS DE DECISIÓN:
───────────────────────
• Confianza del patrón (0.0-1.0)
• Age del patrón (días desde creación)
• Success rate (éxitos / intentos)
• Contexto actual (escena, parámetros)
• Modo de operación (AUTONOMOUS, HYBRID, DEBUG)

9. LIMITACIONES CONOCIDAS
═══════════════════════════════════════════════════════════════════════════════

⚠️ NO PUEDE HACER:

❌ Interfaz Gráfica:
   └─ Solo acepta lenguaje natural o comandos API
   └─ No tiene UI visual en Blender (REPL)

❌ Scripting Complejo:
   └─ No puede escribir código Python nuevo
   └─ Solo ejecutar acciones predefinidas

❌ Animaciones:
   └─ Sin soporte para keyframes
   └─ Sin timeline manipulation

❌ Física:
   └─ Sin simulaciones Fluid/Smoke
   └─ Sin rigid body dynamics

❌ Rendering Avanzado:
   └─ Sin compositor
   └─ Sin post-processing

❌ Otros Engines:
   └─ Solo Blender (no Maya, C4D, etc.)

❌ Persistencia:
   └─ Sesión no se guarda automáticamente
   └─ Cierre = pierde datos de sesión

❌ Decisiones Autónomas:
   └─ Nunca ejecuta sin validación
   └─ Nunca aprende sin confirmación
